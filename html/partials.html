Partials

Partial templates, or just partials, are chunks of code that live in there own file and can be inserted into a page and rendered as if they were written on the page. Partials allow you to break you code up into more manageable chunks. Another benefit of partials is that they can be reused on the same page or in other pages without having to rewrite the code. They are a great way to DRY up your code.

Layout Partials

It is very common to make partials for the header and the footer in the application layout.

layouts/application.html.erb
            
              <body>
                <%= render 'layouts/header' %>
                <%= yield %>
              </body>
layouts/_header.html.erb
            
              
                <div id="header" class="nav navbar navbar-fixed-top">
                  <ul class="brand nav navbar-nav pull-left">
                    <li><%= link_to 'Brand Name', root_url %></li>
                  </ul>
                  <ul class="nav navbar-nav pull-right">
                    <li><%= link_to 'Login', new_session_path %></li>
                  </ul>
                </div>

Form partials

Another very common place to use partials is for displaying forms. If the code for the "new user" form will be the same as the "edit user" form you can write it as a partial and render it on both pages so you don't have to repeat yourself.

users/new.html.erb
            
              <h1>New User</h1>
              <form action="/contacts" method="post">
                <%= render('form') %>
                <button>Create User</button>
              </form>
users/edit.html.erb
            
              <h1>Edit User</h1>
              <form action="/contacts/<%= @contact.id %>" method="post">
                <input name="_method" type="hidden" value="patch">
                <%= render('form') %>
                <button>Update Contact</button>
              </form>
users/_form.html.erb
            
              
                <label for="contact_name">Name</label>
                <input id="contact_name" name="contact[name]" type="text" value="<%= @contact.name %>">
                <label for="contact_phone">Phone</label>
                <input id="contact_phone" name="contact[phone]" type="text" value="<%= @contact.phone %>">
                <label for="contact_email">Email</label>
                <input id="contact_email" name="contact[email]" type="text" value="<%= @contact.email %>">

Notice that you name partials with a leading underscore "_form"(to distinguish them from other views) but when you call the partial "render 'form'.
